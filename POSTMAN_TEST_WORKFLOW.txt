LIMOSUD CARS - POSTMAN TEST WORKFLOW
MOBILE APP STYLE API DOCUMENTATION
================================================================

Purpose:
This file documents every endpoint currently implemented in routes/api.php for Postman/manual API testing.
It is grouped by module and uses the same readable structure requested for the mobile app documentation style.

Base URL:
http://127.0.0.1:8000/api

Authentication:
- Public: no token required.
- Admin: Laravel Sanctum bearer token required.
- Client: no dedicated client-auth routes currently exist.

Common JSON Headers:
Accept: application/json
Content-Type: application/json

Common Admin Headers:
Accept: application/json
Content-Type: application/json
Authorization: Bearer {{admin_token}}

Multipart Admin Headers:
Accept: application/json
Authorization: Bearer {{admin_token}}

Postman Environment Variables:
base_url=http://127.0.0.1:8000/api
admin_email=<ADMIN_EMAIL from .env>
admin_password=<ADMIN_PASSWORD from .env>
admin_token=<saved from POST /api/admin/auth/login>
vehicle_brand_id=<saved from vehicle brand create/list>
vehicle_category_id=<saved from vehicle category create/list>
vehicle_id=<saved from vehicle create/list>
customer_id=<saved from customer create/list>
customer_document_id=<saved from customer document upload>
location_id=<saved from location create/list>
reservation_id=<saved from reservation create/list>
payment_id=<saved from payment create/list>
contract_id=<saved from contract generate/get>
maintenance_id=<saved from maintenance create/list>
expense_id=<saved from expense create/list>
alert_id=<saved from alert create/list>

Common Error Structures:
401 Unauthorized:
```json
{"message":"Unauthenticated."}
```

403 Forbidden:
```json
{"message":"Forbidden."}
```

404 Not Found:
```json
{"message":"No query results for model [App\\Models\\Vehicle] 999"}
```

422 Validation Error:
```json
{"message":"The given data was invalid.","errors":{"field":["Validation message."]}}
```

500 Server Error:
```json
{"message":"Server Error"}
```

5. GET /api/public/vehicles/{vehicle}/availability  [PUBLIC]
   ================================================================

Purpose:
Checks availability for one active vehicle.

Authentication:
Public.

Headers:
Accept: application/json

Query Params:
start_datetime: required date/datetime, for example 2026-08-01 10:00:00.
end_datetime: required date/datetime, must be after start_datetime.

URL Params:
vehicle: numeric vehicle ID.

Request Body:
None.

Success Response:
✅ Captured from Newman run on 2026-06-12.
```json
{"vehicle_id":1,"available":true}
```

Error Responses:
404 if the vehicle does not exist or is inactive.
422 if start_datetime/end_datetime are missing, invalid, or end_datetime is before start_datetime.
500 for unexpected server errors.

Notes:
Availability checks confirmed and in_progress reservations as blocking reservations.
